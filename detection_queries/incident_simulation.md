# Incident Simulation Report

## Project Purpose

This report documents controlled attack simulations performed against the lab environment to validate endpoint logging, centralized monitoring, and detection capability.

The objective is to confirm that security events generated by attack activity are captured by Sysmon, forwarded through Splunk Universal Forwarder, and detected using Splunk queries.

## Simulation 1: Failed Authentication Attack

### Attack Description

A repeated login attempt was performed using incorrect credentials against a domain-connected host.

This simulates:

* password guessing
* brute-force behavior
* credential misuse

---

### Detection Source

Windows Security Event:

```text id="r6m2na"
4625
```

Meaning:

Failed logon attempt

---

### Splunk Query Used

```spl id="t8w4kc"
index=winlogs EventCode=4625
| stats count by Account_Name, Source_Network_Address, host
| where count > 5
```

---

### Expected Detection Result

* repeated failed logons
* source IP identified
* target account identified

---

### Security Impact

If repeated at scale, this may indicate:

* brute-force attack
* password spraying
* account lockout attempt

---

## Simulation 2: PowerShell Execution Detection

### Attack Description

PowerShell was executed on a monitored workstation.

This simulates:

* administrative misuse
* attacker command execution
* script-based payload launch

---

### Detection Source

Sysmon Event:

```text id="n5k7pd"
Event ID 1
```

Meaning:

Process creation

---

### Splunk Query Used

```spl id="u3x9qa"
index=winlogs EventCode=1 Image="*powershell.exe"
| table _time host User Image CommandLine ParentImage
```

---

### Expected Detection Result

* PowerShell process visible
* command line captured
* parent process visible

---

### Security Impact

PowerShell is frequently abused for:

* payload execution
* privilege escalation
* lateral movement

---

## Simulation 3: Lateral Movement Activity

### Attack Description

A remote network logon was generated between systems.

This simulates:

* SMB movement
* remote administration
* credential pivoting

---

### Detection Source

Windows Security Event:

```text id="w1d6lm"
4624
```

Logon Type:

```text id="f7q2yb"
3
```

Meaning:

Successful network logon

---

### Splunk Query Used

```spl id="e4p8tn"
index=winlogs EventCode=4624 Logon_Type=3
| stats count by Account_Name, Workstation_Name, Source_Network_Address, host
```

---

### Expected Detection Result

* remote source visible
* target host visible
* user account identified

---

### Security Impact

This may indicate:

* lateral movement
* administrative pivoting
* unauthorized remote access
